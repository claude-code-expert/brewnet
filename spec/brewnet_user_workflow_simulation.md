# 📖 단계별 상세 시뮬레이션

## [STEP 0] 설치 (5분)

### 시나리오: 사용자가 Brewnet 홈페이지 방문

> **사용자**: "아, 홈서버를 쉽게 구성할 수 있다고? 한번 설치해보자"

### 0.1 설치 옵션 이해

사용자가 마주치는 설치 옵션들:

```bash
# 옵션 1: 편리한 One-liner (추천)
curl -fsSL https://get.brewnet.io/install | bash

# 옵션 2: npm (개발자 친화적)
npx brewnet init

# 옵션 3: Homebrew (macOS)
brew install brewnet

# 옵션 4: 직접 GitHub Release 다운로드
# https://github.com/brewnet/brewnet-cli/releases
```

### ⚠️ 흠결 #1: 설치 옵션이 너무 많음
- **문제**: 사용자가 어느 것을 선택해야 할지 혼동
- **권장사항**: 
  - 명확한 추천 옵션 제시
  - 각 옵션의 장단점 설명
  - 플랫폼별 자동 감지 (curl 설치 스크립트에서)

### 0.2 시스템 체크 (자동)

사용자가 설치 명령어를 실행하면:

```
╔══════════════════════════════════════════════════════════════════╗

║    GitHub: https://github.com/brewnet/brewnet-cli                ║
║                                                                  ║

시스템 요구사항 확인 중...

  ✓ OS: macOS 15.1 (Apple Silicon M4 Pro)
  ✓ 메모리: 24GB RAM (권장: 8GB)
```

### ⚠️ 흠결 #2: 실패 시나리오에 대한 가이드 부족
- **문제**: Docker가 없으면? Git이 없으면? 자동 설치되는가?
- **현상**:
  ```
  ✗ Docker: 설치되지 않음
  
  설치하시겠습니까? (Y/n)
  ```
  - 사용자가 "Y"를 선택하면?
  - 자동으로 Docker를 설치해주나?
  - 아니면 링크를 제공하고 수동 설치를 지시하나?

- **권장사항**:
  - 필수 도구: 자동 설치 지원
  - 선택 도구: 설치 링크 제공
  - 설치 완료 후 재확인 (루프 처리)

   - 다음 링크를 자동으로 브라우저에서 열기
   - OAuth 연동으로 도메인 자동 등록

   - 셋업 완료 후 가이드 제공
   - 각 서비스별 초기 설정 안내

6. **프로젝트 템플릿 저장/로드**

   - 같은 설정으로 재설치 가능

## 최종 평가

### 💚 강점

1. **개념이 명확함** - 인터랙티브 선택형이 직관적
2. **종합적임** - 완전한 홈서버 스택을 한 번에 구성
3. **오픈소스 중심** - 모든 서비스가 자유 라이선스
4. **외부 접근 지원** - Cloudflare Tunnel로 포트포워딩 불필요

### 🤔 개선 필요

1. **선택 혼동** - 초보자를 위한 프리셋/가이드 부족
2. **자동화 미흡** - Cloudflare 도메인 등록이 수동
3. **정보 품질** - 각 선택의 영향과 의미 설명 부족
4. **완료 후 가이드** - 셋업 후 사용법이 불명확

### ✅ 최종 권장사항

**현재 상태**: 개념과 구조는 좋으나 UX 폴리싱 필요

**개발 전 개선**:
1. 프리셋 3개 추가 (미디어/개발/홈자동화)
2. 각 선택지마다 설명 강화
3. Cloudflare 자동 로그인 시도
4. 셋업 완료 후 "다음 단계" 가이드

**개발 중 테스트**:
- 3명 이상의 초보 사용자로 사용성 테스트
- 각 단계에서 중단 지점 분석
- 에러 시나리오 검증

**MVP 릴리즈 후**:
- 실제 사용자 피드백 수집
- 메트릭 기반 개선 (완료율, 소요 시간)
